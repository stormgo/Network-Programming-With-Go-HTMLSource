Cloud computing

Introduction

Cloud computing is the latest trend to come along, labelled by companies such as Gartner as a distruptive phenomenon . In one sense it is just out-sourcing in a different guise, but as such it has particular advantages and disadvantages over earlier outsourcing solutions.

Anyone who uses an email service such as Yahoo or Google mail is already a "cloud user" although at the time they started using it, it probably wasn't called that. These email services encapsulate many of the features of cloud systems:

  • There is a single "point of entry", for example http://gmail.google.com
  • It offers a web-based interface; other common access methods are apps for Apple or Android systems using some protocol (probably HTTP anyway)
  • Yahoo and Google have hundreds of thousands of servers, but the user doesn't care about that: all that is required is that the service can scale up to manage as many users as is needed to the limits of the hardware available
  • Users don't have to manage any of Yahoo's or Google's infrastructure, and as much as is possible, neither do these companies engineers: the software will scale up as needed
  • The infrastructure that powers Google Mail also powers Google Docs, Google Slides, Google Sheets and so on. That is, the infrastructure allows many kinds of services to run and - within some important limits - is agnostic to the application that can be run

This chapter curveys the current state of cloud services, with particular reference to network programming. Later chapters look at how Go can be used in cloud computing

"Something" as a service

There are a number of different ways of accessing "the cloud". These are generally referred to "something as a service" where there are a (growing) number of possible somethings. Currently they include

  • Infrastructure as a service (IaaS)
  • Platfom as a service (PaaS)
  • Software as a service (SaaS)
  • Network as a service (NaaS)

Infrastructure as a service (IaaS)

IaaS basically offers "computers" as a service. Most likely these are virtual machines offered through a hypervisor such as KVM, Xen or VMWare EXSI. A variety of operating systems can be installed on these machines, and they can be further customised by installing applications or configuring the system.

In that sense, IaaS just offers a machine that can be configured to run applications of the programmer's choice. The programming models discussed in previous chapters are not affected by this: what is offered is maintenance, scalability, etc of the server environment.

Examples include Amazon EC2 and Google Compute Engine.
